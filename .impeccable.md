## Design Context

### Users

Primary audience is pragmatic developers evaluating a CLI tool. They land on grrep.com from a search result, a HN comment, or a README link. They give the page roughly ten seconds before deciding to install, bookmark, or close the tab. They are unimpressed by marketing copy, allergic to hero gradients, and can spot a templated dev-tool landing page instantly.

What they need to walk away with, in order:

1. What grrep is (recursive grep that traverses gitjoin boundaries)
2. The install command, copyable
3. One demonstration that proves the claim ("small and fast")
4. A link to the GitHub repo

Everything else on the page is bonus content for the small subset who keep scrolling.

### Brand Personality

Three words: **terse, structural, mechanical.**

- **Terse** — the tool's tagline is one sentence. The site should feel like the tool: minimal config, no ceremony, gets out of the way.
- **Structural** — composition does the heavy lifting. Hierarchy comes from layout and type, not from decoration, color washes, or icons-above-headings.
- **Mechanical** — honest about being a developer tool, but without LARPing as a terminal. No fake CRT effects, no full-page monospace, no `> _` blinking cursors. A well-set Unix man page, not a Hollywood hacker scene.

The interface should evoke the feeling of opening a clean, well-organized tool's documentation — the quiet confidence of something that does one thing precisely.

### Aesthetic Direction

**Terminal-honest, ink-on-paper.** Pick the structural rigor of a printed reference manual and apply it to a 2026 web page. Plain type carries the page; monospace appears only where it earns its place (commands, code, file paths, version metadata).

Concrete moves:

- Asymmetric, left-aligned composition. No centered hero. No symmetric three-column feature grid.
- Type-driven hierarchy: a confident display face for the title, a workhorse text face for body, a precise mono for code. None of them from the banned-reflex list.
- Generous whitespace as structure, not as filler. Vertical rhythm should feel deliberate enough that a reader notices it without being able to articulate why.
- One restrained accent color. Used rarely. It earns attention by appearing twice on the page, not twenty times.
- System-respecting theme: both light and dark get equal craft. Neither is a tinted port of the other — chroma, contrast, and accent placement should be tuned independently for each.

### Anti-references (do NOT look like)

- Generic "developer tool" landing pages with a dark hero, a gradient blob, a cyan-to-purple title, and a row of identical feature cards.
- Inter + dark + neon accent. The default dev-tool monoculture.
- Monospace everywhere as shorthand for "technical."
- Glassmorphism, sparklines as decoration, big rounded icons above every heading, side-stripe borders on cards.
- Hero metric template (giant number, small label, supporting stats).

### Design Principles

1. **Ten-second contract.** Tagline and install command must be readable and copyable within ten seconds. If a design choice slows that down, it loses.
2. **Hierarchy from structure, not chrome.** Type scale, spacing, and alignment do the work. Borders, shadows, gradients, and icons are guilty until proven necessary.
3. **Mono is a tool, not a costume.** Use monospace for things that are actually code or data. Set everything else in a real text face.
4. **Both themes are first-class.** Light and dark each get bespoke palettes, not one derived from the other. Tint neutrals toward the brand hue in OKLCH.
5. **Restraint signals confidence.** One bold gesture (typography, layout, or a single accent moment) surrounded by quiet. The page should feel like the tool feels: small, fast, opinionated.

### Constraints

- Hugo static site. Templates live in the imported `github.com/bep/gitjoin.com` module — design changes touch either the module's templates or local overrides under `layouts/`, `assets/`, etc.
- Sibling site `gitjoin.com` shares the theme module. Changes that affect the shared module need to either be neutral across both or expose enough configuration that grrep can diverge.
